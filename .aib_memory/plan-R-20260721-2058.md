## Goal

Create `ai-critique.md` as a 3,000–5,000-word, evidence-labeled strategic review of the personal website that gives equal weight to professional identity, projects/utilities, and creative work, then presents a prioritized comprehensive-redesign direction compatible with the existing static no-build architecture. The review serves the site owner and a later implementation agent and is grounded in the authoritative current workspace plus representative live observations of routes linked from primary navigation.

## Constraints

- Treat the current workspace as authoritative for both scope and behavior; treat differing live observations as deployment evidence, not as a replacement authority.
- Review deployed routes linked from primary navigation only.
- Create only `ai-critique.md` among website artifacts; do not modify existing HTML, CSS, JavaScript, media, configuration, or content files.
- Keep recommendations strategic, using filenames selectively as evidence rather than providing a file-oriented implementation plan or replacement code.
- Retain static GitHub Pages operation with no mandatory build step, framework, package manager, transpiler, server-side code, database, or custom backend.
- Apply a WCAG 2.2 AA-informed assessment without certification claims and preserve professional contactability while recommending reduced personal-data exposure.
- Discuss external services only as optional enhancements to a backend-independent core.
- Recommend messaging strategy rather than exhaustive replacement copy.

## Success criteria

- `ai-critique.md` exists and contains approximately 3,000–5,000 words.
- `ai-critique.md` contains every section required by the original review request and gives professional, project, and creative purposes equal strategic weight.
- Every material finding in `ai-critique.md` is labeled as workspace-confirmed, live-observed, source-inferred, or subjective judgment.
- `ai-critique.md` covers strategy, information architecture, visual design, UX, WCAG 2.2 AA-informed accessibility, public messaging, privacy, technical quality, performance/reliability, search/sharing readiness, portfolio presentation, and creative presentation.
- Each major recommendation in `ai-critique.md` states the problem, importance, expected benefit, effort, trade-offs, and relationship to the wider redesign direction.
- Recommendations are prioritized by expected user impact relative to effort and distinguish quick wins, foundations, strategic initiatives, and refinements.
- Projects are distinguished as experiments, utilities, completed projects, or personal tools, and creative-content recommendations describe an intentionally curated static gallery.
- Deterministic document checks and editorial review find no unsupported source claims, formal conformance claims, implementation code, or instructions to modify existing website artifacts.
- `git diff -- ai-critique.md` shows only creation of `ai-critique.md` among website artifacts.
