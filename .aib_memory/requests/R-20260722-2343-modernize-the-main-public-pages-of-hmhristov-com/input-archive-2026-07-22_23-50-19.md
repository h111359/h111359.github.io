## Input

### Goal

Modernize the main public pages of hmhristov.com into a coherent, professional, accessible, secure, and responsive personal career website.

The redesigned site must primarily present Hristo M. Hristov as a senior data and analytics leader who also builds useful personal tools and creates visual artwork.

The result must remain a static GitHub Pages website implemented with plain HTML, CSS, and vanilla JavaScript.

### Primary User Value

The redesign must enable professional visitors to quickly understand:

* Hristo’s seniority and professional focus.
* His principal areas of experience and capability.
* The two vocabulary applications he has created.
* His visual artwork.
* How to contact him professionally.

The redesign must improve credibility, readability, navigation, visual consistency, mobile usability, accessibility, and search-and-sharing presentation.

### In-Scope Files

The implementation scope includes:

* `index.html`
* `cv.html`
* `apps.html`
* `art_drawing.html`
* `css/main.css`

The implementation may add small shared static support files where necessary, including:

* A shared JavaScript file such as `js/site.js`.
* Optimized derivative image files for web presentation.
* Other minimal static assets directly required by the redesign.

### Out of Scope

The following work is explicitly excluded:

* Do not modify any file or directory under `202507_china/`.
* Do not link to or promote the China gallery from the redesigned pages.
* Do not redesign either vocabulary application.
* Do not redesign the GitHub Markdown reader, checklist page, links utility, or other tools.
* Do not add the additional utilities to the Projects page.
* Do not introduce a framework, package manager, transpiler, build system, database, backend, authentication system, or server-side code.
* Do not add analytics, contact-form services, external font providers, or new third-party JavaScript dependencies.
* Do not rename the existing public HTML files or change their URLs.
* Do not make a formal WCAG conformance claim.

### Information Architecture

Use the following visible primary-navigation labels:

* Home
* Professional
* Projects
* Creative

Map them to the existing URLs:

* Home → `index.html`
* Professional → `cv.html`
* Projects → `apps.html`
* Creative → `art_drawing.html`

The current page must have a clear visual and semantic active state.

Core navigation must be generated or enhanced through one shared JavaScript component to reduce duplicated maintenance.

Each page must also contain minimal hard-coded semantic navigation that remains functional before JavaScript loads or when JavaScript is unavailable.

Navigation destinations must use ordinary HTML links and must not depend exclusively on JavaScript event handling.

### Homepage

Redesign `index.html` as the main editorial gateway to the site.

The first screen must contain:

* Hristo’s name.
* A concise first-person professional proposition.
* The current portrait presented in a professional responsive composition.
* A primary LinkedIn contact action.
* A secondary email contact action using the current email address.
* Clear navigation toward the Professional, Projects, and Creative pages.

The homepage must lead with Hristo’s professional identity.

Below the main professional introduction, include three equally sized preview sections or cards for:

* Professional
* Projects
* Creative

The three previews must appear equally intentional, although the professional identity remains the principal message of the page.

Homepage text must use first-person narrative voice.

Existing factual meaning must be preserved while grammar, clarity, concision, and professional English are improved.

### Professional Page

Redesign `cv.html` as a concise résumé-style professional profile rather than a complete chronological employment archive.

The page must:

* Use third-person narrative voice.
* Lead with a concise executive summary.
* Present major capabilities, experience themes, selected outcomes, education, skills, languages, certifications, and other relevant professional evidence.
* Prioritize leadership, data and analytics, architecture, organizational development, delivery, and hands-on technical depth.
* Consolidate repetitive material aggressively.
* Permit omission of older, repetitive, or low-value details to maintain a concise résumé.
* Preserve the factual meaning of every retained statement.
* Add no invented achievements, metrics, dates, qualifications, technologies, responsibilities, or claims.
* Avoid implying that a certification or skill has been independently verified as current.
* Remove the existing full detailed chronology rather than placing it in a hidden, collapsible, or secondary archive.

The page must include a dedicated print layout that produces a clean résumé-style result without navigation, animation, decorative controls, or unnecessary page furniture.

### Projects Page

Redesign `apps.html` as a professional Projects page while retaining only the two existing vocabulary applications.

Do not add other utilities or the China gallery.

Each application must be presented with a clear and visually consistent project card containing only information supported by the existing site content, such as:

* Project name.
* Concise purpose.
* Intended user or learning use.
* Brief distinguishing characteristics.
* Clear link to open the application.

The Projects page may use neutral editorial wording where first-person or third-person narration would be unnatural.

The vocabulary applications themselves must remain unchanged.

### Creative Page

Redesign `art_drawing.html` as a curated-looking but complete presentation of the existing artwork.

The page must:

* Display every currently presented artwork.
* Group works by medium.
* Correct visible English and spelling errors.
* Preserve original artwork proportions.
* Avoid arbitrary cropping.
* Use a responsive gallery layout.
* Provide meaningful alternative text.
* Use captions only where supported by existing information.
* Use first-person introductory or contextual text.
* Avoid linking to the China gallery.

The original artwork files must remain unchanged.

Optimized derivative versions may be added and used for public display.

### Visual Direction

Use a restrained warm editorial visual system.

The system must include:

* Warm paper-like background surfaces.
* Dark, high-legibility text.
* One principal earthy accent.
* A restrained secondary surface color.
* Generous whitespace.
* Clear typographic hierarchy.
* Consistent content widths.
* Consistent spacing, borders, focus indicators, buttons, cards, navigation, and footer treatment.
* Professional rather than decorative presentation.
* Pillar-specific layouts that still belong to one shared design system.

Do not use:

* Generic corporate résumé styling.
* Skill meters.
* Dashboard-style visualizations.
* Stock illustrations.
* Excessive gradients.
* Decorative visual clutter.
* External webfonts.

Use high-quality system font stacks only.

### Responsive Design

Target current evergreen versions of:

* Chrome
* Edge
* Firefox
* Safari
* Current mobile browsers based on those engines

The redesigned pages must remain usable at narrow mobile widths and large desktop widths.

Content must:

* Reflow without horizontal page scrolling at approximately 320 CSS pixels.
* Use readable line lengths.
* Use responsive spacing and typography.
* Preserve image aspect ratios.
* Avoid layout shifts where practical by specifying image dimensions or aspect ratios.
* Provide appropriately sized touch targets.

### Interaction and Motion

Use visible but controlled entrance and scroll-based animations for sections and cards.

Animations must:

* Reinforce hierarchy rather than distract from content.
* Avoid blocking navigation or reading.
* Avoid making content unavailable before animation completion.
* Use progressive enhancement.
* Respect `prefers-reduced-motion`.
* Be disabled or substantially reduced when reduced motion is requested.
* Avoid introducing animation-specific external libraries.

Hover styling must not be the only indication of interactivity.

All interactive elements must have visible keyboard focus states.

### Accessibility

Apply a WCAG 2.2 AA-informed accessibility baseline to all four redesigned pages.

Implementation and manual review must cover:

* Valid semantic HTML.
* A declared page language.
* One clear primary heading per page.
* Logical heading hierarchy.
* Semantic landmarks.
* A skip-navigation link.
* Native links and buttons for their intended purposes.
* Keyboard-operable navigation and controls.
* Visible keyboard focus.
* Meaningful link text.
* Informative image alternative text.
* Decorative-image handling where applicable.
* Sufficient text and component contrast.
* Responsive reflow.
* Text-spacing resilience.
* Appropriate touch-target sizing.
* Reduced-motion support.
* No reliance on color alone to communicate meaning.
* Appropriate page titles.
* Correct table semantics if any data tables remain.

The redesign must not claim certified WCAG 2.2 AA conformance without specialist testing.

### Navigation Resilience

Each redesigned page must include usable hard-coded navigation in its HTML.

The shared JavaScript navigation component may replace, synchronize, or enhance that markup after loading.

If JavaScript fails:

* Primary navigation must remain visible and usable.
* Page content must remain readable.
* Contact links must remain usable.
* No essential content may depend on scroll-animation scripts becoming available.

### Metadata and Search Presentation

Each redesigned page must have unique and accurate:

* `<title>`
* Meta description
* Canonical URL
* Open Graph title
* Open Graph description
* Open Graph URL
* Open Graph type
* Social-preview metadata appropriate to the page

Metadata must accurately reflect visible page content.

Do not use keyword stuffing or claims not present in the page.

Use representative existing or optimized local images for social previews where suitable.

### Contact and Privacy

Use LinkedIn as the primary professional contact method.

Retain the current email address as a secondary `mailto:` contact link.

Do not add a contact form.

Review the redesigned pages for unnecessary exposure of personal information, but do not remove professional facts solely because they are public unless their removal follows the Professional-page condensation rule.

External links that open a new browsing context must use appropriate safe relationship attributes.

### Static-Site Security

Apply proportionate static-site security hardening.

The implementation must:

* Add no secrets, tokens, private keys, credentials, or private configuration.
* Add no new third-party scripts.
* Avoid unsafe inline event-handler attributes.
* Avoid creating new HTML injection points.
* Avoid rendering untrusted content through `innerHTML`.
* Use safe external-link behavior.
* Preserve correct URL protocols.
* Avoid misleading contact or submission interfaces that imply backend processing.
* Keep all new functionality compatible with static GitHub Pages hosting.

Existing trusted gallery behavior under `202507_china/` is outside this scope and must not be modified.

### Image Performance

Preserve all original image files.

Create optimized derivative images where this materially improves loading performance.

Optimized derivatives should:

* Use practical web dimensions.
* Use compression appropriate to the artwork or portrait.
* Preserve visual quality and aspect ratio.
* Have stable filenames and predictable organization.
* Be used through suitable HTML image markup.
* Include explicit dimensions or aspect-ratio information.
* Use lazy loading for below-the-fold images where appropriate.
* Avoid lazy-loading the principal above-the-fold portrait when it would delay the primary visual.

Do not modify files under `202507_china/`.

### Content Editing Rules

Preserve the semantic meaning of the existing text while improving English, clarity, hierarchy, and concision.

The implementation may:

* Rewrite sentences.
* Correct grammar and spelling.
* Consolidate repeated ideas.
* Reorder content.
* Replace weak headings.
* Convert dense text or tables into more appropriate semantic structures.
* Omit older or repetitive professional details from the shortened Professional page.

The implementation must not:

* Invent facts.
* Invent quantitative impact.
* Invent current certification status.
* Invent employers, dates, roles, technologies, projects, awards, or education.
* Change the meaning of retained professional claims.
* Present subjective promotional claims as objective facts.

### Shared Components

The redesign should use reusable patterns for:

* Header.
* Navigation.
* Footer.
* Buttons and text links.
* Section introductions.
* Cards.
* Contact actions.
* Image galleries.
* Active navigation state.
* Scroll-reveal behavior.
* Reduced-motion behavior.

The shared JavaScript must remain small, understandable, and non-essential to basic navigation and content access.

1. `index.html`, `cv.html`, `apps.html`, and `art_drawing.html` use one coherent warm editorial design system.

3. Existing URLs and filenames remain unchanged.
4. The navigation works with JavaScript enabled.
5. A usable navigation fallback remains present when JavaScript is unavailable.
6. The homepage leads with Hristo’s senior data and analytics leadership identity.
7. The homepage contains equal follow-on previews for Professional, Projects, and Creative.
8. The current portrait remains present and has appropriate alternative text.
9. LinkedIn is the primary contact action.
10. The current email address remains available as a secondary `mailto:` link.
11. `cv.html` is a concise résumé-style page and no longer contains the complete detailed chronology.
12. Retained professional facts are not materially altered.
13. No unsupported professional claims or metrics are introduced.
14. `apps.html` presents exactly the two existing vocabulary applications.
15. The vocabulary applications themselves are unchanged.
16. `art_drawing.html` displays all currently included artworks grouped by medium.
17. Original artwork proportions are preserved.
18. The redesigned pages do not link to the China gallery.
19. No file under `202507_china/` is modified.
