## Goal

Give the China gallery owner a backward-compatible, non-destructive way to control public media visibility while retaining Delete for every item, and keep the local editor synchronized through remembered folder/event restoration plus safe manual and five-second foreground refresh. The impacted components are `202507_china/data_editor.html`, `202507_china/app.js`, and `.aib_memory/context.md`.

## Constraints

- The site and editor must remain static, dependency-free, offline-capable vanilla HTML and JavaScript with no server or build tooling.
- The File System Access API cannot silently grant access to `./202507_china/data`; startup may restore only a previously selected handle whose permission remains granted, otherwise the existing Open folder action remains available.
- Media uses optional boolean `visible`; only `visible: false` is hidden, so legacy or malformed missing/non-false values remain visible.
- Visibility applies only to image/video records. Text blocks do not serialize or honor it.
- Delete remains available for every media and text item; visibility is an additional native checkbox labeled “Visible.”
- Refresh reads only the selected event file, not `events.js`, and compares complete source text before rerendering.
- Manual refresh confirms before discarding dirty changes. Automatic refresh defers changed disk content while dirty, resumes after a clean transition, pauses in hidden tabs, and checks immediately on return.
- Refresh errors preserve the last valid editor state and use non-blocking feedback.
- The successful all-hidden gallery state displays the exact message “No items.”
- Existing trusted rich-text `innerHTML` behavior, registry editing, media ordering, and base `name`, `preview`, and `desc` fields must remain intact.

## Success criteria

- Every media card in `202507_china/data_editor.html` displays both the retained Delete action and a native “Visible” checkbox; text cards retain Delete and have no visibility checkbox.
- Media loaded without `visible` appears checked, media checkbox edits mark the editor dirty, and Save writes a boolean `visible` field for media only without losing other edits.
- `202507_china/app.js` omits only media whose `visible` value is exactly false and never hides text blocks because of that property.
- A successfully loaded event with no displayable normalized entries renders “No items” rather than a load error.
- A previously chosen directory and last selected event restore on startup when permission is already granted; unavailable, stale, denied, or unsupported restoration leaves the Open folder flow usable.
- Manual refresh of a dirty editor applies only after confirmation; cancellation preserves the current in-memory state.
- Foreground polling checks the selected event at five-second intervals without overlapping reads, rerenders only changed full text, defers replacement while dirty, applies after the editor becomes clean, pauses while hidden, and checks immediately when visible again.
- Missing, renamed, empty, malformed, or unreadable selected files retain the last valid editor data and show a non-blocking error.
- Dependency-free static checks pass, `git diff --check` reports no whitespace errors, and the Chromium manual acceptance flow passes.
