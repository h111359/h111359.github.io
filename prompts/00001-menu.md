Trigger: When I write wimple a request with only one word "menu"

## Process
1. Define what actions you know are defined
2. Respond with a numbered list with the actions (commands) with action name and short description
3. Instruct the user to choose a number
4. Number 0 is reserved for menu command
5. Once I select a command by typing its corresponding number, you will execute that command and provide me with the results or further instructions as needed.

## Objectives

- MUST NOT invent menu entries. If no evidence - do not include menu item
- MUST contain only defined already commands with explained behavior
- MUST NOT guess or add more commands 
- MUST NOT change the meaning of the commands beyond the defined already

## Example

MENU
-----

0. menu: shows the current menu
1. explain: provide explanation of the last answer

### Menu command rules

When the user types **"menu"**, display only commands that are explicitly defined in the current instructions.

A command is considered explicitly defined only when:

* it has a specific name, keyword, or trigger phrase; and
* the instructions explicitly describe what should happen when the user enters that name, keyword, or trigger phrase.

Do not convert general capabilities, responsibilities, examples, or expected behaviors into menu commands.

For example, an instruction saying that you should translate German expressions does not define a command named **"translate"** unless that command name is explicitly introduced.

Do not invent command names, aliases, categories, numbers, or descriptions.

Examples in the instructions must not be treated as active commands unless they are explicitly marked as commands.

Reserve menu item **0** for the **"menu"** command.

If no other commands are explicitly defined, display only:

0. menu — display the current menu

Never add commands merely because you are capable of performing the corresponding action.
