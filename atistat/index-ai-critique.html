Резюме

Сайтът е функционален, визуално последователен и представя услугите на АТИСТАТ разбираемо. Въпреки това той все още не достига стандарта на професионален инструмент за привличане на запитвания. Липсващите метаданни ограничават представянето в търсачки и социални мрежи, част от основното съдържание зависи от успешно изпълнение на JavaScript, мобилното оформление създава проблем с главното заглавие, а портфолиото не дава достатъчно доказателства за реализираните проекти.

Предимства

  • Ясна визуална идентичност. Последователната серифна типография, отличителната зелено-кремава палитра и архитектурните изображения изграждат разпознаваем, спокоен и професионален характер. Единният стил помага на посетителя да възприеме отделните секции като части от една цялостна марка.
  • Разбираемо описание на услугите. Проектирането, строителството, инвестиционното управление и свързаните дейности са представени с достъпен език. Посетителят може бързо да разбере какво предлага компанията, без да преминава през прекалено технически или рекламни формулировки.
  • Логична структура на началната страница. Последователността от представяне, опит, услуги и контакти намалява умственото натоварване за нов посетител. Българската и английската версия следват сходен модел, което създава предвидимо преживяване за различните аудитории.

Спешни проблеми

  • Невидимо основно съдържание при проблем със скрипта. Елементи с клас .at-fade започват скрити. Ако JavaScript файлът не се зареди или прекъсне преди активиране на анимацията, важни текстове могат да останат недостъпни, въпреки че самият HTML е наличен.
  • Отрязано заглавие на тесни екрани. Ограничението в ширината на колоната на главната секция кара заглавието да излиза извън видимата област при малки мобилни устройства. Това засяга първото и най-важно послание на страницата.
  • Липсваща основа за откриваемост и споделяне. Страниците нямат уникални описания, Open Graph данни, Twitter Card данни и структурирана информация за организацията. В резултат търсачките и социалните платформи не получават достатъчно контекст за качествено представяне.

Детайлен анализ

Предимства

  • Професионална визуална идентичност с последователни цветове, типография и архитектурна образност.
  • Четима структура на услугите, която позволява бързо ориентиране в обхвата на работа.
  • Работеща двойка от българска и английска версия с ясно превключване между езиците.
  • Фотографиите са предоставени във формат WebP и са с разумни файлови размери за статичен сайт.
  • Достъпна семантична структура на страниците с логична йерархия от H1, H2 и H3 заглавия.

Недостатъци

  • Видимостта на част от съдържанието зависи от JavaScript, защото елементите за плавна поява са скрити по подразбиране.
  • Главното заглавие излиза извън екрана и се отрязва при тесни мобилни ширини.
  • На страниците липсват мета описания и данни за качествено споделяне в социални мрежи.
  • Няма структурирани данни, чрез които търсачките да разпознаят организацията и контактната информация.
  • Страниците на проектите съдържат само заглавие и един ред за услугата, без снимки, контекст, резултати или доказателства за изпълнение.
  • Картите в хронологията на секция „Опит“ не водят към съответните подробни страници на проектите.
  • В основния поток липсва ясен призив за действие, видим бутон за запитване или директни връзки за телефон и имейл.
  • Вградените SVG логотипи в хедъра и футъра използват повтарящи се идентификатори, което нарушава уникалността на HTML идентификаторите.
  • Главното изображение се зарежда отложено, въпреки че е ключово за първия екран, и така забавя първоначалното визуализиране на мобилни устройства.
  • Пренасочването от HTTP към HTTPS и защитата на транспорта са сървърни въпроси извън обхвата на статичните файлови промени, описани по-долу.

Препоръки за подобрение

Всяка задача по-долу включва промпт за AIB brain, който може да бъде подаден директно на AI агента за изпълнение на промяната.

  1. [Важно] Оптимизация на зареждането на главното изображение

    Главното изображение се зарежда с атрибут за отложено зареждане, което причинява забавяне от около четири секунди при мобилни устройства. Замяната му с незабавно зареждане и приоритизиране на ресурса значително ще подобри показателя LCP.

    Виж промпт 
    ## Input
In file `atistat/index.html`, the hero image in the `#hero` section currently has `loading="lazy"`. Change the attribute to `loading="eager"` and add `fetchpriority="high"` to the same `<img>` element. Do not change any other attributes or surrounding HTML.

## Goal
The hero image loads immediately on page load without browser-imposed deferral, reducing mobile LCP from the current ~3.86 s.

  2. [Важно] Видимост на съдържанието при изключен JavaScript

    CSS класът .at-fade скрива елементи по подразбиране. Ако JavaScript не се зареди или срещне грешка, важно съдържание остава невидимо. Видимостта трябва да е базовото поведение, а анимацията — прогресивно подобрение.

    Виж промпт 
    ## Input
In `atistat/wp-content/themes/atistat/assets/css/main.css`, the `.at-fade` rule sets `opacity: 0`. Change the default state so `.at-fade` elements are visible (`opacity: 1`) when no JavaScript class is applied. In `atistat/wp-content/themes/atistat/assets/js/main.js`, add a class (e.g. `js-enabled`) to `<html>` on script initialization, and scope the hide-then-reveal animation to `.js-enabled .at-fade` so the animation only runs when JavaScript is confirmed active.

## Goal
Core page content is visible to all users regardless of JavaScript availability; the fade-in animation is a progressive enhancement applied only when the script executes successfully.

  3. [Важно] Мобилна четливост на главното заглавие

    Главното заглавие на страницата се отрязва вдясно при ширина на екрана 390 px. Причината е минимална ширина на grid елемента, която е по-голяма от наличното пространство.

    Виж промпт 
    ## Input
In `atistat/wp-content/themes/atistat/assets/css/main.css`, locate the CSS rule that sets a `min-width` or a fixed width on the hero grid column containing the H1 element. Change or remove the min-width constraint so the heading wraps correctly at 320–390 px viewport widths. Test the result at 320 px to confirm no horizontal overflow.

## Goal
The hero H1 text wraps and remains fully readable at 390 px viewport width with no horizontal clipping or overflow.

  4. [Важно] Метаданни за търсачки и социални мрежи

    Нито една страница на сайта няма мета описание, Open Graph тагове или Twitter Card метаданни. Без тях търсачките показват непривлекателни фрагменти, а споделянето в социални мрежи не включва описание или изображение.

    Виж промпт 
    ## Input
In `atistat/index.html` and `atistat/index-en.html`, add to the `<head>` section: (1) a `<meta name="description">` tag with a concise Bulgarian description for index.html and an English description for index-en.html; (2) Open Graph tags — `og:title`, `og:description`, `og:type` set to "website", `og:url`, and `og:image` pointing to the site logo or a representative photo; (3) Twitter Card tags — `twitter:card` set to "summary_large_image", `twitter:title`, `twitter:description`. Also add a JSON-LD `<script type="application/ld+json">` block with Organization schema containing the company name, URL, and contact information.

## Goal
Each page has a unique meta description and social sharing metadata; search engine result snippets show a meaningful description; sharing links on social platforms display a title, description, and image.

  5. [Важно] Навигация към страниците на проектите

    Картите на проектите в секция „Опит“ отварят разгъващ се панел, но не съдържат връзка към детайлната страница на проекта. Потребителят попада в задънена улица без достъп до пълната информация.

    Виж промпт 
    ## Input
In `atistat/index.html`, locate the project timeline cards in the `#opit` section. Each card currently uses a `data-href` attribute or a toggle button to expand a panel. Add a real `<a>` link element to each card pointing to the corresponding project page inside `atistat/proekti/` (e.g., `proekti/project-name/`). The link should be visually styled as a "Виж проект →" button using the existing `.at-btn` class and placed at the bottom of the expanded panel.

## Goal
Every project timeline card contains a working link to its detail page; keyboard and screen-reader users can navigate to project details without using JavaScript-only interactions.

  6. [Важно] Призив за действие и форма за запитване

    Сайтът не съдържа ясен призив за действие — няма бутон за запитване, форма за контакт или видим телефонен номер в критичните секции. Потенциалните клиенти нямат очевидна следваща стъпка.

    Виж промпт 
    ## Input
In `atistat/index.html`, add a prominent CTA section after the `#services` section and before `#contacts`. The section should contain a short Bulgarian heading (e.g., "Свържете се с нас"), a one-sentence value proposition, and two call-to-action buttons using the `.at-btn` class: one `<a href="mailto:...">` for email and one `<a href="tel:...">` for phone, with the contact details taken from the existing `#contacts` section of the same page.

## Goal
A visible, keyboard-accessible CTA with email and phone links appears between the services and contacts sections, giving visitors a clear next step without scrolling to the bottom of the page.

  7. [Важно] Съдържание на страниците за проекти

    Шестте страници за проекти съдържат само заглавие и един ред с описание на услуга. Те не демонстрират реална работа и не удовлетворяват очакванията на потенциалния клиент, дошъл от портфолиото.

    Виж промпт 
    ## Input
For each project page in `atistat/proekti/` (there are six pages, each at `atistat/proekti/<project-name>/index.html`): add an `<article>` element with the following structure — a `<h1>` project name, a `<p>` service category, a placeholder `<figure>` with an `<img>` pointing to a representative image (use an existing image from the site if available, or a placeholder path), a `<section>` with a `<h2>` "За проекта" and a `<p>` placeholder for project description text (mark with a HTML comment `<!-- TODO: owner to provide project description -->`), a `<section>` with a `<h2>` "Детайли" and a bullet list of project facts (location, area, year — mark with TODO comments for owner to fill), and a `<a href="../">← Обратно към портфолиото</a>` back link. Reuse the ATISTAT theme CSS and header/footer from `atistat/index.html`.

## Goal
Each project page contains a structured layout ready for the owner to fill with real project facts, images, and descriptions; the page no longer appears as a placeholder; a back-to-portfolio link is present.

  8. [Важно] Уникални идентификатори на SVG елементите

    Хедърът и футърът на сайта съдържат два идентични SVG логотипа с еднакви ID атрибути. Повтарящите се ID стойности нарушават HTML стандарта и могат да причинят проблеми при екранни четци и автоматизирани инструменти.

    Виж промпт 
    ## Input
In `atistat/index.html` and `atistat/index-en.html`, the `<header>` and `<footer>` each contain an inline SVG with the same ID values (`svg1`, `defs1`, `g1`, `layer2`, `layer3`, `namedview1`, `text1`, `text2`). Rename the IDs in the footer SVG by appending `-footer` to each ID and updating any internal `href` or `url()` references within the same SVG element. Do not change the header SVG IDs.

## Goal
All element IDs in the document are unique; HTML validation passes without duplicate-ID errors; assistive technologies can unambiguously reference each SVG element.
